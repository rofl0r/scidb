<h2>Scidb (<abbr>.sci</abbr>)</h2>

Questo è il formato proprio di Scidb. L'uso di questo formato è fortemente raccomandato, dato che le funzioni di Scidb sono pienamente supportate solo in questo formato e fornirà un'ottima velocità nell'uso.

<h2>Scid (<abbr>.si4</abbr>, <abbr>.si3</abbr>)</h2>

Questo è il formato database di Scid e data la sua ampia distribuzione è supportato sia in lettura sia in scrittura. Usando questo formato non sarà possibile usare tutte le funzioni disponibili in Scidb.

<h2>ChessBase (<abbr>.cbh</abbr>)</h2>

L'azienda ChessBase mantiene questo formato segreto, per questo solo la lettura è supportata. Non tutti i dati della partita possono essere letti correttamente, con alcune partite apparirà il messaggio che comunica la non riuscita dell'operazione di decodifica.

<h2>PGN (<abbr>.pgn</abbr>, <abbr>.pgn.gz</abbr>, <abbr>.zip</abbr>)</h2>

Il formato <abbr>PGN</abbr> (Portable Game Notation) è uno standard comune per lo scambio di partite di scacchi. L'apertura di un file <abbr>PGN</abbr> è permessa in sola lettura. Se si hanno grandi collezioni di partite in questo formato si consiglia di convertirle nel formato di Scidb.

<!-- NEW
<p>
A complete specification of this format will be found at

Standard: Portable Game Notation Specification and Implementation Guide</a>.
</p>

<h2>Database archive (<abbr>.sca</abbr>)</h2>

<p>
Scidb's database archive is based on an independent archive format,
and contains one (or perhaps several) databases in archived form. This
facilitates the shipment of databases through the internet, because it
is a single file instead of several ones. This database archive is not
limited to a specific database format.
</p>

<p>
A specification of this format is on page
<a href="Database-Archive.html">Database archive</a>.
</p>
