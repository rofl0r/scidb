TITLE Formati Database

<h2><img src="../images/ft-scidb.png" alt="ft-scidb" /> Scidb (<abbr>.sci</abbr>)</h2>

Questo è il formato proprio di Scidb. L'uso di questo formato è fortemente raccomandato, dato che le funzioni di Scidb sono pienamente supportate solo in questo formato e fornirà un'ottima velocità nell'uso.

<p>
I database Scidb sono sempre codificati con il set caratteri UTF-8. Per questo
i database possono essere usati senza complicazioni su tutti i sistemi e con tutte
le localizzazioni.
</p>

<p>
A database in Scidb's format is divided into several files:
</p>

<box>
  <table style="margin-top: 0.5em; margin-bottom: 0.5em;">
    <tr>
      <td nowrap="nowrap" valign="top"><kbd>.sci =&nbsp;</kbd></td>
      <td>Game data</td>
    </tr>
    <tr>
      <td nowrap="nowrap" valign="top"><kbd>.scg =&nbsp;</kbd></td>
      <td>Game moves, commentaries, annotations, move information, and PGN tags</td>
    </tr>
    <tr>
      <td nowrap="nowrap" valign="top"><kbd>.scn =&nbsp;</kbd></td>
      <td>Player/Event/Site/Annotator data</td>
    </tr>
  </table>
</box>

<h2><img src="../images/ft-scid.png" alt="ft-scid" /> Scid (<abbr>.si4</abbr>, <abbr>.si3</abbr>)</h2>

Questo è il formato database di Scid e data la sua ampia distribuzione è supportato sia in lettura sia in scrittura. Usando questo formato non sarà possibile usare tutte le funzioni disponibili in Scidb.

<annotation>
I database Scid saranno aperti in modalità sola-lettura, l'utente può passare alla modalità
in scrittura dopo l'apertura. L'ultima modalità di utilizzo sarà preservata alla prossima apertura.
</annotation>

<p>
In alcuni casi l'utente deve scegliere la codifica appropriata nell'aprire il
database, per assicurare una corretta visualizzazione dei nomi e di altro testo.
I database più recenti potrebbero avere codifica UTF-8, database più vecchi potrebbero
essere scritti con  il set di caratteri Latin-1(ISO-8859-1), il formato Scid non fornisce
informazioni sulla codifica utilizzata. In alcuni casi tuttavia l'identificazione
automatica del set caratteri dovrebbe avere successo.
</p>

<p>
Questo formato ricorda per struttura il formato Scidb:
</p>

<box>
  <table style="margin-top: 0.5em; margin-bottom: 0.5em;">
    <tr>
      <td nowrap="nowrap" valign="top"><kbd>.si4 =&nbsp;</kbd></td>
      <td>Dati partita</td>
    </tr>
    <tr>
      <td nowrap="nowrap" valign="top"><kbd>.sg4 =&nbsp;</kbd></td>
      <td>Mosse, commenti, annotazioni e tag PGN</td>
    </tr>
    <tr>
      <td nowrap="nowrap" valign="top"><kbd>.sn4 =&nbsp;</kbd></td>
      <td>Dati relativi a Giocatore/Evento/Luogo/Turno</td>
    </tr>
    <tr>
      <td nowrap="nowrap" valign="top"><kbd>.ssc =&nbsp;</kbd></td>
      <td>Dati di ordinamento</td>
    </tr>
  </table>
</box>

<h2><img src="../images/ft-chessbase.png" alt="ft-chessbase" /> ChessBase (<abbr>.cbh</abbr>)</h2>

L'azienda ChessBase mantiene questo formato segreto, per questo solo la lettura è supportata. Non tutti i dati della partita possono essere letti correttamente, con alcune partite apparirà il messaggio che comunica la non riuscita dell'operazione di decodifica.

<annotation>
I database ChessBase non possono essere usati come databases di riferimento.
</annotation>

<p>
Generalmente, questo database ha codifica con set caratteri CP1252, ma in alcuni casi
una selezione manuale del set caratteri durante l'apertura del database può essere
necessaria, per assicurare una corretta visualizzazione dei nomi e di altro testo.
</p>

<p>
Il formato database di ChessBase è diviso in diversi file separati, quelli più importanti sono:
</p>

<box>
  <table style="margin-top: 0.5em; margin-bottom: 0.5em;">
    <tr><td nowrap="nowrap" valign="top"><kbd>.cbh &nbsp;=&nbsp;</kbd></td><td>Dati partita</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.cbg &nbsp;=&nbsp;</kbd></td><td>Mosse e varianti</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.cba &nbsp;=&nbsp;</kbd></td><td>Commenti e annotazioni</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.cbs &nbsp;=&nbsp;</kbd></td><td>Indice delle fonti</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.cbp &nbsp;=&nbsp;</kbd></td><td>Indice giocatori</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.cbc &nbsp;=&nbsp;</kbd></td><td>Indice commentatori</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.cbt &nbsp;=&nbsp;</kbd></td><td>Indice tornei</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.cbe &nbsp;=&nbsp;</kbd></td><td>Indice squadre</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.cbj &nbsp;=&nbsp;</kbd></td><td>Informazioni squadra</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.ini &nbsp;=&nbsp;</kbd></td><td>File di inizializzazione per il database</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.html =&nbsp;</kbd></td><td>Documento HTML</td></tr>
  </table>
</box>

<h2><img src="../images/ft-pgn.png" alt="ft-pgn" /> PGN (<abbr>.pgn</abbr>, <abbr>.pgn.gz</abbr>, <abbr>.zip</abbr>)</h2>

Il formato <abbr>PGN</abbr> (Portable Game Notation) è uno standard comune per lo scambio di partite di scacchi. L'apertura di un file <abbr>PGN</abbr> è permessa in sola lettura. Se si hanno grandi collezioni di partite in questo formato si consiglia di convertirle nel formato di Scidb.

<p>
Secondo lo standard PGN un file PGN ha codifica Latin-1 (ISO-8859-1).
</p>

<p>
Per le specifiche complete su questo formato visitare
<a href="http://www.saremba.de/chessgml/standards/pgn/pgn-complete.htm">
Standard: Portable Game Notation Specification and Implementation Guide</a>.
</p>

<h2 id="Database-Archive"><img src="../images/ft-archive.png" alt="ft-archive" /> Archivio Database (<abbr>.scv</abbr>)</h2>

<p>
L'archivio database di Scidb è basato su un formato archivio indipendente,
e contiene uno (o più) database nella forma archiviata. Questo facilita
il trasferimento di database su internet, perché si tratta di un singolo file
invece di molti. Questo archivio database non è limitato a uno specifico formato database.
</p>
