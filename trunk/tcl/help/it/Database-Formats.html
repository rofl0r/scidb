Formati Database

Scidb supports following formats:

  • Scidb
  • Scid
  • ChessBase
  • PGN
  • Archivio Database

Scidb (.sci)

Questo è il formato proprio di Scidb. L'uso di questo formato è fortemente raccomandato, dato che le funzioni di Scidb sono pienamente supportate solo in questo formato e fornirà un'ottima velocità nell'uso.

Only Scidb's format (and PGN format) is storing games of various chess variants, currently this is:

Normal Chess (including Scacchi 960, Scacchi Shuffle)
Crazyhouse Chess
Vinciperdi (Suicide, Giveaway)
Losers (variant of Vinciperdi)
Three-check Chess

A database is fixed to one chess variant (the Clipbase is an exception).

I database Scidb sono sempre codificati con il set caratteri UTF-8. Per questo i database possono essere usati senza complicazioni su tutti i sistemi e con tutte le localizzazioni.

A database in Scidb's format is divided into several files:

.sci =  Game data
.scg =  Game moves, commentaries, annotations, move information, and PGN tags
.scn =  Player/Event/Site/Annotator data

Scid (.si4, .si3)

Questo è il formato database di Scid e data la sua ampia distribuzione è supportato sia in lettura sia in scrittura. Usando questo formato non sarà possibile usare tutte le funzioni disponibili in Scidb.
I database Scid saranno aperti in modalità sola-lettura, l'utente può passare alla modalità in scrittura dopo l'apertura. L'ultima modalità di utilizzo sarà preservata alla prossima apertura.

In alcuni casi l'utente deve scegliere la codifica appropriata nell'aprire il database, per assicurare una corretta visualizzazione dei nomi e di altro testo. I database più recenti potrebbero avere codifica UTF-8, database più vecchi potrebbero essere scritti con il set di caratteri Latin-1 (ISO-8859-1), il formato Scid non fornisce informazioni sulla codifica utilizzata. In alcuni casi tuttavia l'identificazione automatica del set caratteri dovrebbe avere successo.

Questo formato ricorda per struttura il formato Scidb:

.si4 =  Dati partita
.sg4 =  Mosse, commenti, annotazioni e tag PGN
.sn4 =  Dati relativi a Giocatore/Evento/Luogo/Turno
.ssc =  Dati di ordinamento

I file con estensione .sso (Scid Search Option File), .sor (Scid Repertoire File), e .stm (Scid Tree Mask File) non appartengono esplicitamente al database e non saranno considerati da Scidb. Likewise any saved game list (file suffix .txt) will be disregarded.

ChessBase (.cbh)

L'azienda ChessBase mantiene questo formato segreto, per questo solo la lettura è supportata. Non tutti i dati della partita possono essere letti correttamente, con alcune partite apparirà il messaggio che comunica la non riuscita dell'operazione di decodifica.
I database ChessBase non possono essere usati come databases di riferimento.

Generalmente, questo database ha codifica con set caratteri CP1252, ma in alcuni casi una selezione manuale del set caratteri durante l'apertura del database può essere necessaria, per assicurare una corretta visualizzazione dei nomi e di altro testo.

Il formato database di ChessBase è diviso in diversi file separati, quelli più importanti sono:

.cbh  = Dati partita
.cbg  = Mosse e varianti
.cba  = Commenti e annotazioni
.cbs  = Indice delle fonti
.cbp  = Indice giocatori
.cbc  = Indice commentatori
.cbt  = Indice tornei
.cbe  = Indice squadre
.cbj  = Informazioni squadra
.ini  = File di inizializzazione per il database
.html = Documento HTML

PGN (.pgn, .pgn.gz, .zip)

Il formato PGN (Portable Game Notation) è uno standard comune per lo scambio di partite di scacchi. L'apertura di un file PGN è permessa in sola lettura. Se si hanno grandi collezioni di partite in questo formato si consiglia di convertirle nel formato di Scidb.

Secondo lo standard PGN un file PGN ha codifica Latin-1 (ISO-8859-1). Ma se un file PGN comincia con un marcatore UTF-8 (UTF-8-BOM: byte sequence EF BB BF) ci si aspetterà un encoding UTF-8 (in osservanza a Unicode FAQ).

Only the following variants will be accepted if reading FICS games:

wild/2   misc/bishops-only   crazyhouse
wild/7   misc/knights-only   endings/kbbk
wild/8   misc/little-game   endings/kbnk
wild/17   misc/no-queens   losers
wild/19   misc/pawns-only   odds/*
wild/22   misc/pyramid   pawns/little-game
wild/23   misc/queen-rooks   pawns/pawns-only
wild/25   misc/queens-only   pawns/wild-five
wild/26   misc/rooks-only   suicide
wild/fr   misc/runaway  

Per le specifiche complete su questo formato visitare Standard: Portable Game Notation Specification and Implementation Guide.

Archivio Database (.scv)

L'archivio database di Scidb è basato su un formato archivio indipendente, e contiene uno (o più) database nella forma archiviata. Questo facilita il trasferimento di database su internet, perché si tratta di un singolo file invece di molti. Questo archivio database non è limitato a uno specifico formato database.

Una dettagliata descrizione di questo formato è alla pagina Specification of database archive.
