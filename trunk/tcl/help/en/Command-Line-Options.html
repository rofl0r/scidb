Kommandozeilenoptionen

On the command line the following options are available at program start:

-- After this option all arguments wil be interpreted as file names. This permits that file names may start with a hyphen.
--help Anzeige der Optionsmöglichkeiten (in Englisch) und sofortiges Programmende.
--show-board Zum Brett-Tab wechseln sofort nach Programmstart.
--fast-load Nur die ausschliesslich erforderlichen Dateien nach Programmstart laden. Dies ermöglicht einen deutlich schnelleren Programmstart. Diese Option sollte i.A. nicht benutzt werden, sie dient vornehmlich dem Testen.
--first-time Das Programm wird gestartet wie beim ersten Mal, d.h. die Benutzeroptionen werden zurückgesetzt, die Wiederherstellungsdateien werden gelöscht, und die Standard-Brettthemen werden erneut in das benutzereigene Verzeichnis kopiert, sofern sie nicht modifiziert wurden. Diese Option kann nützlich sein wenn der Programmstart aufgrund eines Fehlers in der Optionendatei fehlschlägt.
--elo-only Keine weiteren Rating-Dateien laden ausser dem ELO-Rating. Dies beschleunigt den Programmstart.
--print-recovery-files Zeige die Wiederherstellungsdateien, die in der letzten Sitzung aufgrund ungespeicherter Partien erzeugt wurden, und terminiere das Programm. Möglicherweise existieren keine und es werden dann auch keine angezeigt.
--delete-recovery-files Die aus der letzten Sitzung existierenden Wiederherstellungsdateien, die ungespeicherte Partien enthalten, löschen und das Prgramm terminieren. Es erfolgt keine physikalische Löschung, mit der Option --recover-old können sie wiederbenutzt werden.
--dont-recover Das Programm starten ohne Wiederherstellungsdateien, die ungespeicherte Partien enthalten, aus der letzten Sitzung zu laden. Diese Dateien werden nicht physikalisch gelöscht,mit der Option --recover-old können sie wiederbenutzt werden.
--recover-old Ungespeicherte Partien aus älteren Sitzungen wiederherstellen. Die ungespeicherten Partien aus der letzten Sitzung werden nicht mehr berücksichtigt.
--single-process Erzwingt eine weitere Scidb-Instanz bei Programmstart. Diese Option dient nur dem Testen und sollte nicht verwendet werden.
--force-grab Wird das Programm in einem Debugger gestartet werden Grabs automatisch unterdrückt, denn diese können zum Einfrieren des Bildschirms führen. Mit dieser Option werden die Grabs auch im Debugger zugelassen.

Die folgenden Optionen werden von der Tk-Bibliothek erkannt:

-geometry GEOMETRY Benutze die Geometrieangaben für das Hauptfenster. Beispielsweise hat das Hauptfenster mit der Angabe -geometry 600x800+20+10 die Größe 600x800 und wird an der Bildschirmposition mit den Koordinaten 20,10 positioniert.
-display DISPLAY Benutze das mit DISPLAY angegebene Display.
-sync Benutze den Synchron-Modus für den Display-Server.
