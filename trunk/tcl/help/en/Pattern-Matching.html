Pattern matching

A pattern is a string which may contain the following special characters:

* Matches any sequence of characters in string, including a null string.
? Matches any single character in string.
\x Matches the single character x. This provides a way of avoiding the special interpretation of the characters *?\ in pattern.

Pattern matching is always case-insensitive.

Examples

Polgar, J*
This pattern is matching the following strings: "Polgar, J", and "Polgar, Judit".
A?erba?h

This will match the following names: "Averbakh" (English spelling), and "Awerbach" (German spelling).

really intended\?
Is only matching the string "really intended?". The character ? in the pattern does not have a special meaning because it is escaped with Backslash. This means that this pattern is not matching the string "really intended!".
