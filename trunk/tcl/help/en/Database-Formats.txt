TITLE Database formats

<h2><img src="../images/ft-scidb.png"> Scidb (<abbr>.sci</abbr>)</h2>

<p>
This is Scidb's own database format. Use of this format is strongly
recommended because of speed and feature completeness considerations.
</p>

<p>
Scidb databases are always encoded with character set UTF-8. Hence
these databases can be used without any complications under all systems
and locations.
</p>

<p>
A database in Scidb's format is divided into several files:
</p>

<box>
  <table style="margin-top: 0.5em; margin-bottom: 0.5em;">
    <tr>
      <td nowrap="nowrap" valign="top"><kbd>.sci =&nbsp;</kbd></td>
      <td>Game data</td>
    </tr>
    <tr>
      <td nowrap="nowrap" valign="top"><kbd>.scg =&nbsp;</kbd></td>
      <td>Game moves, commentaries, annotations, move information, and PGN tags</td>
    </tr>
    <tr>
      <td nowrap="nowrap" valign="top"><kbd>.scn =&nbsp;</kbd></td>
      <td>Player/Event/Site/Annotator data</td>
    </tr>
  </table>
</box>

<h2><img src="../images/ft-scid.png"> Scid (<abbr>.si4</abbr>, <abbr>.si3</abbr>)</h2>

<p>
Scid's database format is fully supported because of it's wide adoption.
But with this format you cannot use the full capability of Scidb.
</p>

<annotation>
Scid databases will be opened in read-only mode, the user may switch to
writeable mode after opening. The last mode will be prerserved while
re-opening the database.
</annotation>

<p>
In some cases the user has to choose the appropriate encoding while opening
the database, to ensure a correct display of names and other text. Newer
databases may be UTF-8 encoded, older database may be written with Latin-1
(ISO-8859-1) character set, the Scid format does not provide information
about this. But in many cases the automatic detection of the character set
should be successful.
</p>

<p>
This format resembles structurally Scidb's format:
</p>

<box>
  <table style="margin-top: 0.5em; margin-bottom: 0.5em;">
    <tr>
      <td nowrap="nowrap" valign="top"><kbd>.si4 =&nbsp;</kbd></td>
      <td>Game data</td>
    </tr>
    <tr>
      <td nowrap="nowrap" valign="top"><kbd>.sg4 =&nbsp;</kbd></td>
      <td>Game moves, commentaries, annotations, and PGN tags</td>
    </tr>
    <tr>
      <td nowrap="nowrap" valign="top"><kbd>.sn4 =&nbsp;</kbd></td>
      <td>Player/Event/Site/Round data</td>
    </tr>
    <tr>
      <td nowrap="nowrap" valign="top"><kbd>.ssc =&nbsp;</kbd></td>
      <td>Sorting data</td>
    </tr>
  </table>
</box>

<h2><img src="../images/ft-chessbase.png"> ChessBase (<abbr>.cbh</abbr>)</h2>

<p>
The company ChessBase keeps this format secret, therefore only the reading
mode is supported. Not all game data can be read correctly, with some games
the message of failed decoding will appear.
</p>

<annotation>
ChessBase databases cannot be used as reference database.
</annotation>

<p>
Usually this database is encoded with character set CP1252, but in some cases
a manual selection of the the character set while opening the database may
be required, to ensure a correct display of names and other text.
</p>

<p>
A database in ChessBase format is divided into many separate files, the important ones are:
</p>

<box>
  <table style="margin-top: 0.5em; margin-bottom: 0.5em;">
    <tr><td nowrap="nowrap" valign="top"><kbd>.cbh =&nbsp;</kbd></td><td>Game data</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.cbg =&nbsp;</kbd></td><td>Game moves and variations</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.cba =&nbsp;</kbd></td><td>Annotations and commentaries</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.cbs =&nbsp;</kbd></td><td>Source index</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.cbp =&nbsp;</kbd></td><td>Players index</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.cbc =&nbsp;</kbd></td><td>Commentators index</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.cbt =&nbsp;</kbd></td><td>Tournament index</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.cbe =&nbsp;</kbd></td><td>Team index</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.cbj =&nbsp;</kbd></td><td>Team information</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.ini =&nbsp;</kbd></td><td>Database initialisation file</td></tr>
    <tr><td nowrap="nowrap" valign="top"><kbd>.html =&nbsp;</kbd></td><td>HTML document folder</td></tr>
  </table>
</box>

<h2><img src="../images/ft-pgn.png"> <abbr>PGN</abbr> (<abbr>.pgn</abbr>, <abbr>.pgn.gz</abbr>, <abbr>.zip</abbr>)</h2>

<p>
The <abbr>PGN</abbr> format (Portable Game Notation) is a common standard for transfering
chess games. Opening a <abbr>PGN</abbr> file is allowed in reading mode only. With larger
<abbr>PGN</abbr> game collections it is recommended to convert it into the Scidb format.
</p>

<p>
According to the PGN standard a PGN file is encoded with character set 
Latin-1 (ISO-8859-1).
</p>

<p>
A complete specification of this format will be found at
<a href="http://www.saremba.de/chessgml/standards/pgn/pgn-complete.htm">
Standard: Portable Game Notation Specification and Implementation Guide</a>.
</p>

<h2 id="Database-Archive"><img src="../images/ft-archive.png"> Database archive (<abbr>.scv</abbr>)</h2>

<p>
Scidb's database archive is based on an independent archive format,
and contains one (or perhaps several) databases in archived form. This
facilitates the shipment of databases through the internet, because it
is a single file instead of several ones. This database archive is not
limited to a specific database format.
</p>

<p>
A detailed description of this format is on page

</p>
