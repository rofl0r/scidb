Partiemarkierungen

Eine Markierung ("Flag") ist ein Kennzeichen irgendeiner Charakteristik, die für jede Partie der Datenbank ein- oder ausgeschaltet werden kann. Es gibt 21 vom Benutzer einstellbare Markierungen, die direkt in jede Partie gesetzt werden können. Diese Markierungen dienen vornehmlich zum Suchen von Partiedaten, oder zum Filtern von Partielisten. Diese Markierungen haben ansonsten keine weitere Bedeutung und können deshalb vom Benutzer frei verwendet werden.

Die Markierungen und ihre Symbole im einzelnen:

 Schwarze Eröffnung
 Mittelspiel
 Endspiel
 Neuerung
 Bauernstruktur
 Taktik
 Königsseite
 Damenseite
 Brillianz
 Schnitzer
 User
 Beste Partie
 Entscheidungsturnier
 Modellpartie
 Strategie
 Angriff
 Opferspiel

 Material
 Figurenspiel
 Regelwidrige Rochade
 Regelwidriger Zug

Die beiden letzten Markierungen können nicht vom Benutzer eingestellt werden, sie werden automatisch gesetzt wenn eine Partie ungültige Rochaden oder sogar andere ungültige Züge (zum Beispiel der König zieht ins Schach) in der Hauptvariante enthält. Sind beide Markeriungen in einer Partie enthalten so wird nur die Markierung für Regelwidriger Zug angezeigt.

Es gibt noch eine weitere besondere, nicht vom Benutzer einstellbare Markierung, die Lösch-Markierung. Eine Partie mit diesem Flag ist zur Löschung vorgemerkt, und die Partie wird erst entfernt, wenn die Datenbank komprimiert wird (siehe Datenbankumschalter: Komprimieren).

Beim Bearbeiten von Partien in einer Scid-Datenbank (.si4) werden zudem die sechs speziellen Benutzermarkierungen (1..6) unterstützt. Es wird jedoch von einer Verwendung dieser Markierungen abgeraten, es kommt zu nicht lösbaren Problemen beim Kopieren von Partien in eine andere Datenbank; auch die Applikation Scid kann diese Probleme nicht lösen.

Bei der Umwandlung von ChessBase-Datenbanken werden beide Markierungen Strategische Schnitzer und Taktische Schnitzer auf die gemeinsame Markierung Schnitzer abgebildet, Scidb unterscheidet nicht zwischen diesen beiden Typen.
