Schreibschutz einer Datenbank

Eine Datenbank kann schreibgeschützt bzw. schreibberechtigt gesetzt werden. Dies ist nur für Scidb-Datenbanken (.sci) sowie für PGN-Archive (.pgn, .pgn.gz) möglich. Scid- und ChessBase-Datenbanken (.si3, .si4, .cbh, .cbf) sowie Zip-Archive (.zip) sind immer schreibgeschützt.

Ein Schreibschutz hat keine Auswirkungen auf die Schreibberechtigung der Dateien, es handelt sich um ein applikationsspezifisches Merkmal. Beim Laden einer Datenbank kann sich die Applikation an die letzte Einstellung erinnern, sofern diese Datenbank noch in der Historie vorhanden ist.

Es existieren folgende Einschränkungen:

  • Datenbankdateien bzw. PGN-Dateien, die keine Schreibberechtigung besitzen, können nicht schreibberechtigt gesetzt werden. Dazu muß erst eine Änderung der Dateiberechtigungen erfolgen.

  • Wird der Ladevorgang einer PGN-Datei unterbrochen, so wird aus technischen Gründen keine Schreibberechtigung zugelassen.

  • Die Ablage-Datenbank kann nicht schreibgeschützt werden.

  • PGN-Dateien (.pgn, .pgn.gz) werden immer schreibgeschützt geöffnet, sofern nicht in der Historie eine andere Einstellung vorhanden ist. Sie müssen also explizit schreibberechtigt gesetzt werden, wenn eine Bearbeitung erfolgen soll.

Zu beachten ist, daß bearbeitete PGN-Archive explizit gespeichert werden müssen, bei größeren PGN-Archiven kann dies einige Zeit in Anspruch nehmen. Ist ein PGN-Archiv verändert worden, dann erkennt man dies an einer rötlichen Einfärbung des Datenbankpiktogramms.
