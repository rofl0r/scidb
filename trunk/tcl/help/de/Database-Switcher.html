Der Datenbankumschalter

Der Datenbankumschalter ermöglicht den einfachen Wechsel zwischen Datenbanken. Die aktive Datenbank wird durch einen gelben Hintergrund hervorgehoben. Alle Datenbanken, die nicht zur Schachform der aktiven Datenbank gehören, sind gesperrt (weißer Hintergrund).

Mit Halten der mittleren Maustaste (i.d.R. das Mausrad) auf einem Datenbankfeld wird eine Informationsbox angezeigt.

Die Funktionalitäten im Datenbankumschalter sind:

  • Administration
  • Verwaltung
  • Drag & Drop

Administration

Zur Administration einer Datenbank steht ein kontextsensitives Menü bereit:

Eigenschaften Öffnet eine Box mit der Anzeige verschiedener Datenbankinformationen.

Export Öffnet einen Dialog zum Datenbankenexport in andere Datenbankformate oder in ein Dokumentenformat.
Enthält diese Datenbank mehrere Schachformen (nur bei der Ablage-Datenbank und bei PGN-Dateien möglich) dann werden nur die Partien der aktiven Schachform exportiert.

PGN-Dateien importieren Öffnet den Dateiauswahldialog, um die zu importierenden PGN-Dateien auszuwählen. Es können mehrere PGN-Dateien gleichzeitig gewählt werden. Die Partien werden an die bestehende Datenbank angehängt.

Datenbanken importieren Die zu importierenden Datenbanken werden über den Dateiauswahldialog ausgewählt. Die Partien werden an die bestehende Datenbank angehängt. Es können mehrere Datenbanken gleichzeitig ausgewählt werden.

Verwaltung Dieses Untermenü bietet Funktionalitäten um Datenbanken zu verwalten (⇒ Verwaltung).

Archiv erzeugen Ein Datenbankarchiv für die gewählte Datenbank anlegen. Der Dateiauswahldialog wird geöffnet, um den Archivnamen zu wählen.

Schliessen Die gewählte Datenbank wird geschlossen. Sind Partien aus dieser Datenbank bearbeitet, aber noch ungespeichert, erfolgt vor der Schliessung eine Sicherheitsabfrage.
Eine Schliessung der Ablage-Datenbank ist nicht möglich.

Schreibgeschützt Diese Datenbank wird schreibgeschützt bzw. schreibberechtigt gesetzt. Dies ist nur für Scidb- und Scid-Datenbanken (.sci, .si4, .si3) möglich.
PGN-Dateien und ChessBase-Datenbanken sind immer schreibgeschützt.
Die Ablage-Datenbank kann nicht schreibgeschützt werden.

Datenbank öffnen Eine weitere Datenbank öffnen.

Öffne zuvorige Eine Datenbank öffnen über die Datenbankhistorie.

Neue Datenbank anlegen (Normalschach) Eine neue Datenbank anlegen. Es können nur Scidb- und Scid-Datenbanken (.sci, .si4, .si3) neu angelegt werden. Soll die Datenbank Partien einer anderen Schachform speichern dann wird der nächste Menüpunkt verwendet.

Neue Datenbank anlegen Eine neue Datenbank einer anderen Schachform anlegen. In diesem Fall können nur Scidb-Datenbanken neu angelegt werden.

Symbolgröße Auswahl der Symbolgröße für die Datenbankkacheln.

Verwaltung

Dieses Untermenü bietet Funktionalitäten um Datenbanken zu verwalten:

Piktogramm wählen Ein neues Piktogramm für diese Datenbank auswählen.
Dies ist nicht für die Ablage-Datenbank möglich.

Beschreibung bearbeiten Die Beschreibung dieser Datenbank ändern bzw. neu anlegen.
Dies ist nicht für die Ablage-Datenbank möglich.

Zuginformationen entfernen Eine Auswahl von Zuginformationen (wie z.B. "Verbrauchte Zugzeit") aus der Datenbank entfernen. Dies ist beispielsweise auch nützlich für Importe von FICS-Partien, um nachträglich die unerwünschten Informationen (hier "Verbrauchte Millisekunden") zu entfernen.

PGN-Tags entfernen Dieses Funktion durchsucht die Datenbank nach zusätzlichen PGN-Tags (wie "WhiteTeam"). Es kann dann eine Auswahl von ihnen aus der Datenbank entfernt werden. Das Entfernen nicht benötigter Tags beschleunigt ein wenig die Positionssuche, aber erst wenn nach der Entfernung eine Kompaktierung der Datenbank (⇒ Kompaktieren) geschieht.

Kompaktieren Diese Datenbank komprimieren, d.h. die zum Löschen vorgemerkten Partien werden physikalisch gelöscht. Diese Aktion kann nicht rückgägngig gemacht werden. Dieser Menüpunkt ist gesperrt wenn keine Partien in dieser Datenbank mit einer Lösch-Markierung versehen sind.

Drag & Drop

Datenbanken können auch durch Drop (Fallenlassen) einer URI-Liste geöffnet werden. Wird eine URI-Liste (Dateienliste) – oder eine einzelne Datei – in das Datenbankumschaltfenster gezogen, so signalisiert ein farbiger Hintergrund, daß ein Drop erfolgen kann. Auf diese Weise kann das Öffnen von Datenbanken beispielsweise auch mit einem externen Dateimanager erfolgen.

Die Dateienliste kann auch über ein Datenbankfeld fallengelassen werden. In diesem Fall werden die Partien in die Zieldatenbank kopiert.

Um alle Partien einer Datenbank (jedoch nur einer Schachform) in eine andere zu kopieren werden mit gedrückter linker Maustaste die Partien gegriffen und in das Zielfeld gezogen und fallengelassen. Voraussetzung ist, daß die Zieldatenbank nicht schreibgeschützt ist und auch nicht gesperrt. Dies ist ein wichtiges Instrument zum Verwalten und Kopieren von Dateien zwischen Datenbanken.
